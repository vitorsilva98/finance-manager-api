Casos de teste CategoryController

[POST] /api/v1/finance/categories
    ERRO:
        1. Token JWT válido sem prefixo "Bearer"
        2. Token JWT inválido com prefixo "Bearer"
        3. Token JWT expirado com prefixo "Bearer"
        4. Token JWT válido com prefixo "Bearer"
            4.1. Campo "name" vazio
            4.2. Campo "name" nulo
            4.3. Campo "name" com mais de 100 caracteres
            4.4. Campo "name" preenchido com um valor já salvo no banco de dados

    SUCESSO:
        1. Campo "name" preenchido com um valor que não foi salvo no banco de dados e token JWT válido com prefixo "Bearer"

[GET] /api/v1/finance/categories
    ERRO:
        1. Token JWT válido sem prefixo "Bearer"
        2. Token JWT inválido com prefixo "Bearer"
        3. Token JWT expirado com prefixo "Bearer"

    SUCESSO:
        1. Token JWT válido com prefixo "Bearer"

[GET] /api/v1/finance/categories/{id}
    ERRO:
        1. Token JWT válido sem prefixo "Bearer"
        2. Token JWT inválido com prefixo "Bearer"
        3. Token JWT expirado com prefixo "Bearer"
        4. Token JWT válido com prefixo "Bearer"
            4.1. Path variable "id" com id diferente do formato UUID
            4.2. Path variable "id" com id em formato UUID porém sem nenhum registro equivalente cadastrado

    SUCESSO:
        1. Path variable "id" com id em formato UUID com registro equivalente cadastrado e token JWT válido com prefixo "Bearer"

[DELETE] /api/v1/finance/categories/{id}
    ERRO:
        1. Token JWT válido sem prefixo "Bearer"
        2. Token JWT inválido com prefixo "Bearer"
        3. Token JWT expirado com prefixo "Bearer"
        4. Token JWT válido com prefixo "Bearer"
            4.1. Path variable "id" com id diferente do formato UUID
            4.2. Path variable "id" com id em formato UUID porém sem nenhum registro equivalente cadastrado
            4.3. Path variable "id" com id em formato UUID com registro equivalente cadastrado porém usuário não possui ROLE_ADMIN
            4.4. Path variable "id" com id em formato UUID com registro equivalente cadastrado, usuário possui ROLE_ADMIN
                porém categoria já está relacionada com pelo menos um lançamento

    SUCESSO:
        1. Path variable "id" com id em formato UUID com registro equivalente cadastrado, categoria não relacionada com nenhum lançamento,
            usuário possui ROLE_ADMIN e token JWT válido com prefixo "Bearer"
